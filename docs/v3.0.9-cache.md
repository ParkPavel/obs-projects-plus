## Задача 1: Фильтры в создании заметок

### Текущее состояние

Все 4 вида УЖЕ имеют инъекцию фильтров с оператором `is`:

| Вид | Функция | Файл | Строки |
|-----|---------|------|--------|
| Board | `getFilterValues(conditions, excludeField?)` | `BoardView.svelte` | 246-256 |
| Table | `getFilterValuesFromConditions(conditions)` | `TableView.svelte` | 55-62 |
| Gallery | `getFilterValuesFromConditions(conditions)` | `GalleryView.svelte` | 40-47 |
| Calendar | Inline (то же + дата/время/endTime/creationDate) | `CalendarView.svelte` | 1516-1565 |

### Проблемы

1. **Дублирование**: 3 копии одной логики (Table=Gallery, Board отличается excludeField)
2. **Нет CreateNoteContext**: Table/Board/Gallery НЕ передают context в `CreateNoteModal`
   - Calendar передаёт `{ date, time? }` → интерполяция шаблонов `{{date}}` / `{{time}}`
   - В остальных видах дата/время берётся из `dayjs()` (текущее время), но это default behaviour modal
3. **Board**: Корректно исключает groupBy поле из фильтров (чтобы не конфликтовать с колонкой)

### План исправления

1. Вынести `getFilterValuesFromConditions(conditions, excludeField?)` в `src/lib/helpers.ts`
2. Заменить локальные реализации на импорт из helpers
3. Calendar: заменить inline код на вызов той же функции
4. Решение: НЕ добавлять CreateNoteContext в Table/Board/Gallery — текущий default (dayjs()) корректен

## Задача 2: Мгновенные переходы (без анимаций)

### Инфраструктура

**`src/lib/helpers/animation.ts`** (25 строк):
- `getScrollBehavior()` → `'auto'` для instant, `'smooth'` для smooth
- `getAnimationDuration(default)` → `0` для instant, `default` для smooth
- Настройка: `settings.preferences.animationBehavior: "smooth" | "instant"`

### Найденные баги

#### БАГ 1: CSS анимация `.period-container` (КРИТИЧЕСКИЙ)
- **Файл**: `InfiniteHorizontalCalendar.svelte:1043`
- **Проблема**: `animation: periodSlideIn 0.3s cubic-bezier(...)` ВСЕГДА активна
- **Эффект**: Week/day виды анимируются при переключении периодов даже в instant mode
- **CSS `@media (prefers-reduced-motion: reduce)`** уже есть (строка 1253) — но это OS-level, не plugin setting

#### БАГ 2: Хардкод 400ms в NavigationController
- **Файл**: `NavigationController.ts:109,146`
- **Проблема**: `this.animateScroll(container, targetScroll.top, 400)` — жёстко 400ms
- **Должно быть**: `this.animateScroll(container, targetScroll.top, getAnimationDuration(400))`
- animated=true/false не учитывает plugin setting

#### БАГ 3: Нет CSS-класса для instant mode
- **Проблема**: Нет глобального класса `.ppp-instant-mode` для подавления CSS анимаций
- `styles.css` имеет `@media (prefers-reduced-motion)` → подавляет ВСЕ анимации
- Но plugin setting "instant" — это отдельная настройка, не привязанная к OS

### Корректно работает ✅
- `InfiniteHorizontalCalendar.smoothScrollTo()` — использует `getAnimationDuration(400)`
- `InfiniteCalendar.scrollToDate()` — использует `getScrollBehavior()`
- `scrollBy()` — использует `getScrollBehavior()`
- `AnimationController.animate()` — поддерживает `duration <= 0`

### План исправления

1. **NavigationController.ts**: Заменить `400` → `getAnimationDuration(400)` в обоих scrollVertical/HorizontalCalendar
2. **InfiniteHorizontalCalendar.svelte**: Добавить reactive class `ppp-instant-mode` на `.period-container` + CSS `animation: none` когда instant
3. **styles.css**: Добавить правило `.ppp-instant-mode` для подавления всех CSS анимаций/transitions в plugin scope

## 1. Диагноз (почему «функции есть, а пользоваться нельзя»)

Раунды тестирования показывают один паттерн: **каждая поверхность говорит на своём
диалекте**. Таблица уже говорит на Notion-грамматике (после TABLE_V2_CANON), но панель
чарта — «простыня» из 15 контролов, настройки блока — «— inherit from view —» без
объяснений, конвейер — инженерные термины, вкладки — «New View ×3». Notion-качество — это
не отдельные красивые виджеты, а **одна грамматика на весь продукт**.

## 2. Интерфейсная грамматика (5 примитивов Notion)

Всё взаимодействие в Notion собрано из пяти примитивов. У нас есть компоненты для каждого —
рециклинг означает: каждая поверхность ОБЯЗАНА использовать их и только их.

| Примитив | Notion-паттерн | Наш компонент | Правило |
|---|---|---|---|
| **Menu** | поповер-список: иконка+подпись, секции, submenu chevron, danger внизу | `contextMenu.ts` / `FloatingPopup`+`PopoverList` | любой список действий. Никаких самодельных меню |
| **Panel** | боковой peek / попап-секции с аккордеонами; одна колонка «label → control» | `SlideInPanel`, `WidgetConfigShell` | любые настройки. Анатомия из §3 |
| **Pill / Chip** | цветной токен значения (select, фильтр, связь) с × и +N | pills из Table V2 → вынести в `_shared/Pill.svelte` | любое значение-токен везде (фильтры, теги, связи, сорт) |
| **Inline edit** | клик по значению = редактор на месте, blur=commit, Esc=revert | `EditableCell` (паттерн) | любое значение редактируется там, где показано |
| **Hover-reveal** | управление невидимо до hover (⋮⋮, ↗, ⊕, ⋯) | реализовано в Shell/TableRow | управление никогда не занимает место постоянно |

**Tone of voice (D1, Vision §7)**: каждая подпись — на языке задачи с примером.
Не «inherit from view», а «Данные этого вью (по умолчанию)». Не «standalone», а
«Без связи — блок показывает все записи». Словарь — в дизайн-сессии Wave 2.

## 3. Канон панели настроек (анатомия — закрывает #090)

```
┌─ Panel ────────────────────────────────┐
│ Заголовок                          ✕  │
│ ── Секция «Данные» ────────────── ▾ ──│   ← аккордеон, 2–4 секции максимум
│  Подпись поля        [контрол]        │   ← строка: label слева, контрол справа
│  объяснение-пример мелким текстом     │   ← D1: «например, выручка по клиенту»
│ ── Секция «Вид» ────────────────── ▸ ─│   ← продвинутое свёрнуто по умолчанию
│                          [Готово]     │
└────────────────────────────────────────┘
```
Правила: слайдер всегда показывает значение; select всегда имеет человечные опции;
«(нет)» запрещено — вместо него «Не выбрано — будет X»; deprecated-контролы не показываются.

## 5. Волны (глобальный пайплайн исполнения)

Каждая волна = коммиты + 4 ворот + деплой-верификация + ручной чек по своему чек-листу.

**W1 — «Таблица и представления до конца»** (механика управления):
F2.4 меню колонки (Sort/Hide/Freeze/Calculate▸/Edit property через contextMenu),
resize колонок, `[+]` add property (существующий CreateField-флоу);
F2.5 группировка (TableGroupSection) + #088 вкладки (тип при создании, имена, меню вкладки);
#081 RelationPickerPopover → редактор Relation-ячейки.

**W2 — «Язык и панели»** (дизайн-сессия → реализация):
словарь tone-of-voice (один MD: термин → подпись → пример); #090 анатомия §3 на все
панели; #092+#075 конвейер (recovery + empty-state «было→стало»); filter-pills в
ControlBar каждого вью; #060 tooltips; #071/#089 round-trip конфигов.

**W3 — «Связи и формулы»**: #091 guided link-flow; #077 единый FormulaConstructor-попап
(5 точек входа, anatomy-схема); #076 sub-base входы.

**W4 — «Первый опыт»**: #061 пресеты (4 из visual stack) + профили (3 из Vision);
#082 typed-карточка; SmartSuggest → discovery-канал W1–W3 функций.

**W5 — «Фундамент»**: #078 CalendarView decomposition; #079 hex-ratchet; #036 mobile.

Гейты пользователя (вне волн): #066 (YAML-конфиг), #080 (Formula Node), push/merge.

## 6. Контроль качества рециклинга

- Инварианты-ratchet: R0_3 px, R0_4 archive, R0_5 text, R0_6 LOC (+каждый новый компонент).
- Новое правило W2: **запрещены строки UI вне `t()`** в панелях, проходящих рециклинг.
- Чек-лист волны = скриншоты «до/после» по поверхностям волны (папка screanshots — канал).
- Definition of Done волны: пользователь не может отличить грамматику поверхности от
  грамматики таблицы (один язык меню/панелей/пиллов/hover).

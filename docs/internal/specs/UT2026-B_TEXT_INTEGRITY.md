# UT2026-B — Text Integrity: контракт целостности UI-текста

> Домен: #069 (порча кодировки), i18n-часть #071 (CoverBanner хардкод), missing ru-ключи.
> Статус: СОГЛАСОВАНО, реализуется немедленно.

## Природа проблемы

В UI три класса текстовой деградации с одной причиной — **нет машинного контроля текстового слоя**:

1. **Mojibake в исходниках**: 25 литералов `�`/`??` в 4 файлах — эмодзи, разрушенные коммитом
   с битой кодировкой. Прошли через 4 ворот незамеченными, потому что ни один gate не смотрит
   на содержимое строковых литералов.
2. **Сырые i18n-ключи в UI**: `views.dashboard.pipeline.unnest` показан пользователю —
   ключ без `defaultValue` и без записи в ru.json. i18next честно вернул ключ.
3. **Хардкод-английский мимо i18n**: CoverBannerConfig (Image source/Width/Fit/Done) —
   строки не проходят через `$i18n.t` вообще.

Это та самая «поверхностность»: текст добавляется тем способом, который быстрее в моменте,
и нет ни конвенции, ни инварианта, который заставил бы единый способ.

## Контракт

**T1. Ноль U+FFFD и `??`-плейсхолдеров в `src/`.**
Enforcement: jest-инвариант `R0_5_textIntegrity.test.ts` — скан src на `�`; ratchet-список
для legacy-исключений пуст с момента внедрения.

**T2. Каждый пользовательский текст = `t(key, { defaultValue })`.** defaultValue — английский,
ru.json — обязательный перевод, uk/zh — best effort (прецедент #065). Ключ без defaultValue
запрещён в UI-слое (то, что показал pipeline.unnest).

**T3. Иконки — только Lucide через `<Icon name=...>`.** Эмодзи/юникод-глифы в роли иконок
запрещены (расширение инварианта #047 с «не эмодзи» до «только Icon-компонент»).
Включая архив? Нет — архив чинится только от mojibake (T1 глобален), эмодзи в архиве умирают
с #074 (см. UT2026-A).

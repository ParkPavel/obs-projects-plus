# 📖 Руководство пользователя Projects Plus

Добро пожаловать в Projects Plus! Это подробное руководство поможет вам максимально эффективно использовать возможности управления проектами в Obsidian.

## 🚀 Начало работы

### Первые шаги

1. **Включение плагина**
   - Перейдите в Настройки → Сообщество плагинов
   - Включите "Projects Plus"
   - Плагин появится в ленте

2. **Открытие Projects Plus**
   - Нажмите на значок Projects Plus в ленте
   - Или используйте `Ctrl+P` → "Projects Plus: Показать Projects Plus"

3. **Создание первого проекта**
   - Нажмите кнопку "+"
   - Выберите тип проекта
   - Настройте параметры
   - Начните организовывать!

3. **2 недели** — промежуточный вид
4. **Неделя** — детальный недельный план
5. **День** — timeline на один день (07:00–22:00)

## 📱 Мобильная версия (v2.2.0+)

### Особенности мобильного интерфейса

- **Увеличенные ячейки дней** — на 100% больше для удобного касания
- **Плавающие кнопки** — полупрозрачные, минималистичные
- **Плавающая кнопка "Сегодня"** — появляется при скрытом тулбаре
- **iOS Safe Area** — учёт notch/Dynamic Island (v3.1.0+)

### Навигация по заметкам (v3.0.10+)

На мобильных устройствах для навигации используется long-press (500ms):
- **Board, Table, Gallery**: долгое нажатие → контекстное меню с вариантами открытия
- **EditNote**: кнопка «Открыть заметку» → dropdown с 3 режимами (текущая вкладка / новая вкладка / новое окно)

### Мобильная адаптация Agenda (v3.1.0+)

- **Drag-and-drop через grip** — перетаскивание списков только за grip-иконку (⠿), не конфликтует с прокруткой
- **Перемещение списков** — также через контекстное меню «Переместить вверх/вниз»
- **Модальные окна** — отображаются как bottom-sheet на полную ширину
- **Изоляция скролла** — скролл внутри модалей не каскадирует в Obsidian

### 🔀 Drag & Drop — единая система (v3.1.0 / v3.2.0)

Начиная с v3.1.0, все перетаскивания в Projects Plus работают через **grip-иконки** (⠿) — специальные маленькие точки-индикаторы, видимые при наведении (десктоп) или всегда (мобильные). Это гарантирует, что скролл и DnD не конфликтуют.

#### 🖥️ Десктоп — как пользоваться DnD

**Доска (Board):**
1. Наведите курсор на карточку или заголовок колонки — появится grip (⠿)
2. Зажмите grip и перетащите карточку в нужное место / колонку

**Agenda (боковая панель календаря):**
1. Наведите курсор на заголовок списка — появится grip (⠿)
2. Зажмите grip и перетащите список вверх/вниз

**Календарь Timeline (v3.2.0+):**
1. Переключитесь в режим **Неделя** или **День** (timeline с временной шкалой)
2. Зажмите EventBar (цветную полосу события) и перетащите:
   - **Вверх/вниз** → изменение времени (привязка к 15-мин интервалу)
   - **Влево/вправо** → перенос на другой день (в режиме Неделя)
3. Для изменения длительности: наведите на **левый или правый край** многодневной полосы → появится resize-маркер → перетащите

#### 📱 Мобильные — как пользоваться DnD

**Доска и Agenda:**
1. **Долгое нажатие (500ms)** на grip-иконке (⠿) — устройство завибрирует
2. Не отпуская, перетащите элемент в нужное место
3. Отпустите палец для завершения

**Календарь Timeline (v3.2.0+):**
1. Переключитесь в режим **Неделя** или **День**
2. **Долгое нажатие (500ms)** на EventBar — устройство завибрирует, полоса поднимется и станет полупрозрачной
3. Не отпуская, перетащите:
   - **Вверх/вниз** → изменение времени (вибрация при каждом snap к 15-мин интервалу)
   - **Влево/вправо** → перенос на другой день
4. Для resize: **долгое нажатие на край полосы** → потяните для изменения длительности
5. Отпустите палец — событие сохранится в новой позиции

> **Важно:** DnD на мобильных работает **только через grip** (Board/Agenda) или **долгое нажатие** (Calendar). Простое касание всегда открывает событие, а не перетаскивает его — случайные перетаскивания исключены.

### Board: Pinch-to-zoom (v3.0.10+)

Двухпальцевый жест для масштабирования доски:
- Диапазон: 25% — 200%
- Поддержка Safari GestureEvent
- Работает параллельно с Ctrl+Scroll на десктопе

### Галерея: Адаптивная сетка (v3.0.10+)

- Карточки автоматически ограничены шириной 200px на мобильных
- Тактильная обратная связь (`:active` состояния) при касании

### DayPopup — обзор дня

При одинарном тапе на ячейку дня открывается полноэкранный popup:

| Элемент | Действие |
|---------|----------|
| ✅ **Чекбокс** | Изменить статус заметки (выполнено/не выполнено) |
| ⚙️ **Шестерёнка** | Открыть модальное окно редактирования полей |
| 📋 **Дублирование** | Скопировать заметку на выбранные даты |
| 🗑️ **Удаление** | Удалить заметку |

### Дублирование заметок

1. Тапните на кнопку "Дублировать" рядом с заметкой
2. Откроется мини-календарь
3. Выберите одну или несколько дат (тапами)
4. Нажмите "Дублировать"

Заметка будет скопирована на все выбранные даты с сохранением всех полей.

### Скрытие тулбара

- Нажмите кнопку сворачивания в тулбаре (стрелка вверх)
- Тулбар полностью скроется
- Появятся плавающие кнопки в левом верхнем углу
- На мобильных появится плавающая кнопка "Сегодня"

## 📊 Типы представлений

### 📋 Табличное представление
Идеально подходит для проектов с большим количеством данных и метаданных.

**Функции:**
- Сортируемые столбцы

**Лучше всего для:**
- Исследовательских проектов
- Календарей контента
- Управления задачами
- Анализа данных

### 📌 Доска (Kanban)
Интерфейс в стиле Kanban для управления рабочими процессами.

**Функции:**
- Перетаскивание карточек
- Настраиваемые столбцы
- Фильтрация карточек
- Отслеживание прогресса

**Лучше всего для:**
- Agile разработки
- Создания контента
- Рабочих процессов задач
- Фаз проекта

### 📅 Календарное представление
Визуализация проектов на основе временной шкалы.

**Функции:**
- Месячные/недельные представления
- Фильтрация по датам
- Визуализация временной шкалы
- Отслеживание дедлайнов

**Лучше всего для:**
- Планирования мероприятий
- Планирования контента
- Управления дедлайнами
- Временных проектов

#### Поля дат в календаре

Календарь использует три типа полей дат:

| Поле | Назначение | Влияет на позицию |
|------|-----------|:-----------------:|
| `startDate` | Начало события — определяет, на какой день попадёт событие в календаре | ✅ |
| `endDate` | Конец события — для многодневных событий (отображается как полоса) | ✅ |
| `date` | Дата создания заметки — заполняется автоматически при создании | ❌ |

> **Важно:** Поле `date` — это дата создания заметки, а **не** дата начала события. Оно заполняется автоматически при создании заметки через календарь и не влияет на позиционирование события. Для позиционирования события в календаре используйте `startDate`.

## 🔧 Типы проектов

### 📁 Проекты на основе папок

Создавайте проекты из существующих папок в вашем хранилище.

**Настройка:**
1. Щелкните правой кнопкой мыши по любой папке в проводнике файлов
2. Выберите "Создать проект в папке"
3. Настройте параметры проекта
4. Начните организовывать!

**Конфигурация:**
- **Путь**: Расположение папки
- **Рекурсивность**: Включать подпапки
- **Шаблоны**: Шаблон заметки по умолчанию
- **Исключения**: Файлы для игнорирования

### 🏷️ Проекты на основе тегов

Организуйте проекты с помощью тегов Obsidian.

**Настройка:**
1. Создайте новый проект
2. Выберите "Тег" как источник данных
3. Введите ваш тег (например, `#project/my-project`)
4. Настройте параметры иерархии

**Функции:**
- Поддержка иерархических тегов
- Автоматическое обнаружение заметок
- Фильтрация на основе тегов
- Динамические обновления

### 🔍 Проекты Dataview

Используйте запросы Dataview для сложной фильтрации проектов.

**Настройка:**
1. Создайте новый проект
2. Выберите "Dataview" как источник данных
3. Введите ваш запрос
4. Протестируйте и доработайте

**Примеры запросов:**
```dataview
FROM "Projects/MyProject"
WHERE status != "completed"
SORT file.ctime DESC
```

## 📝 Шаблоны

### Базовая структура шаблона

```markdown
---
title: "{{title}}"
status: "draft"
startDate: {{date}}
date: {{date}}
project: "{{project}}"
---

# {{title}}

## Обзор
<!-- Добавьте обзор вашего проекта здесь -->

## Задачи
- [ ] Задача 1
- [ ] Задача 2
- [ ] Задача 3

## Заметки
<!-- Добавьте ваши заметки здесь -->

## Ресурсы
<!-- Добавьте ссылки и ресурсы здесь -->
```

### Продвинутые функции шаблонов

#### Динамические переменные
- `{{title}}` - Заголовок заметки
- `{{date}}` - Текущая дата
- `{{time}}` - Текущее время
- `{{project}}` - Название проекта
- `{{author}}` - Ваше имя

#### Условное содержимое
```markdown
{{#if status}}
Статус: {{status}}
{{/if}}

{{#unless completed}}
Эта задача еще не завершена.
{{/unless}}
```

## 🔄 Рабочие процессы

### Рабочий процесс создания контента

1. **Планирование**
   - Создайте проект в календарном представлении
   - Настройте календарь контента
   - Определите вехи

2. **Создание**
   - Используйте доску для этапов контента
   - Отслеживайте прогресс с помощью карточек
   - Управляйте черновиками и обзорами

## 📚 Продвинутые функции

### Интеграция API

Projects Plus предоставляет богатый API для разработчиков:

```javascript
// Получить все проекты
const projects = app.plugins.plugins['obs-projects-plus'].api.getProjects();

// Создать новый проект
app.plugins.plugins['obs-projects-plus'].api.createProject({
  name: "My Project",
  dataSource: { kind: "folder", config: { path: "/MyProject" } }
});
```

### Пользовательские представления

Создавайте пользовательские представления для специальных нужд:

```javascript
// Регистрация пользовательского представления
app.plugins.plugins['obs-projects-plus'].api.registerView({
  id: "my-custom-view",
  name: "My Custom View",
  component: MyCustomComponent
});
```

### Автоматизация

Используйте Templater или другие плагины для автоматизации:

```javascript
// Скрипт Templater для создания проекта
const project = await tp.user.createProject({
  name: tp.file.title,
  template: "Project Template"
});
```
